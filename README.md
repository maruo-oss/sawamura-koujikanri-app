# 工事管理システム（GAS Webアプリ）

Google Apps Script（GAS）で構築された建設業向け工事管理Webアプリケーションです。  
Google スプレッドシートをデータベースとして利用し、案件・契約・発注・請求・支払などの工事業務を一元管理します。

## 主な機能

### 工事管理
- 案件の一覧表示・作成・編集・削除（論理削除 + カスケード削除対応）
- 工事の登録・進捗率管理・ロック機能
- 案件ステータス別サマリー（進行中 / 商談中 / 契約済 / 完了）

### 契約協議
- 契約協議書の作成・編集・承認フロー
- 契約協議書からの案件・工事自動登録（通常 / 追加工事対応）
- Google スプレッドシートテンプレートによる契約協議書出力（通常 / 小口）
- Excel エクスポート（大工事用 / 小口用）
- **PDF エクスポート（大工事用 / 小口用）— 出力PDFの右上に「出力日付-枝番」（例: `20260428-1`）を自動印字**
  - 枝番は契約協議書ごとに当日初出力時に確定し、同日内の再出力では同じ番号を維持（別契約は +1 で発番）
  - 発番状況は専用シート「PDF出力履歴」で管理（同時押下時の重複発番は `LockService` で抑止）
  - 印字位置はテンプレート側で `<<[出力番号]>>` プレースホルダーセルを置いた位置に決まる
- 入力画面のセクション順序：基本情報 → 発注者情報 → 工事情報 → 担当者情報（上から下に入力できる導線）

### 発注管理
- 発注の作成・編集
- 見積もり詳細の管理（複数明細対応）
- 注文協議書 / 注文書の出力（Google ドキュメントテンプレート方式）
- 発注からの支払データ自動生成

### 請求・支払管理
- 請求書の作成・ステータス管理
- 入金履歴の記録・管理
- 支払の作成・承認フロー・実行記録
- 支払履歴の管理
- 協力会社ごとの支払条件自動適用（手形比率・サイト等）

### 決算集計
- 月次・期別の集計表示

### 実行予算
- 案件別の予算管理・実績集計

### 工事台帳
- 工事台帳の作成・ステータス管理
- 工事に紐づく契約・発注・請求・支払・予算データの集約表示

### スケジュール管理
- ガントチャート表示
- マイルストーン管理（予定日 / 実績日 / 遅延検知）
- カレンダー表示

### マスタ管理
- 顧客マスタ
- 協力会社（業者）マスタ
- 社員マスタ

### その他機能
- CSVインポート（各シートへのデータ一括取込）
- アーカイブ（論理削除データの閲覧・復元・完全削除）
- 操作履歴（監査ログ）
- 添付ファイル管理（Google Drive連携）
- ドキュメントプレビューモーダル

## 技術構成

| 項目 | 技術 |
|------|------|
| バックエンド | Google Apps Script |
| フロントエンド | Vue.js 3（CDN版） |
| データストア | Google スプレッドシート |
| ファイル管理 | Google Drive |
| テンプレート出力 | Google スプレッドシート / Google ドキュメント（プレースホルダー置換方式） |

## ファイル構成

| ファイル | 説明 |
|----------|------|
| `Code.gs` | メインモジュール。Webアプリのエントリポイント（`doGet`）、各種API関数、権限管理、帳票出力、ダミーデータ生成 |
| `Repository.gs` | データアクセス層。各シートへのCRUD操作、論理削除、監査ログ記録を提供する Repository パターン |
| `index.html` | シングルページのフロントエンド。Vue.js 3 による SPA（CSS / JS をすべて内包） |
| `setup_template.gs` | 契約協議書テンプレート（小口用）のセル修正スクリプト |
| `check.gs` | テンプレートスプレッドシートのセル内容確認用デバッグスクリプト |

## アーキテクチャ

```
[ブラウザ]
  │  Vue.js 3 SPA (index.html)
  │  google.script.run でバックエンド呼び出し
  ▼
[Google Apps Script]
  │  Code.gs   ... API層（ビジネスロジック + 権限チェック）
  │  Repository.gs ... データアクセス層（Repository パターン）
  ▼
[Google スプレッドシート]       [Google Drive]
  各管理シート（14+シート）       テンプレート / 出力ファイル / 添付ファイル
```

## 権限管理

4段階のロールベースアクセス制御（RBAC）を実装しています。

| ロール | レベル | 権限概要 |
|--------|--------|----------|
| `readonly` | 1 | 閲覧のみ |
| `staff` | 2 | 自部門データのCRUD |
| `manager` | 3 | 承認権限、削除権限、アーカイブ管理 |
| `admin` | 4 | 全機能アクセス（マスタ管理、完全削除等） |

社員マスタのメールアドレスと Google アカウントを照合してユーザーを識別します。

| 出力名 | 内容 | 形式 | 操作場所 |

## ⚠ 経理が使う帳票は未整備

「注文書発行」までは出力が揃っていますが、その先の **請求書受領 → 査定 → 確定 → 振込 → 通知** のフローで以下が未整備です。

- **請求書 PDF 原本の添付・管理** — 業者から受け取った請求書を登録時に紐付ける機能がない
- **全銀フォーマット（FBデータ）出力** — インターネットバンキング用の振込データを出せない
- **業者向け月次支払額決定通知書** — PDF 生成・郵送・オンライン通知すべて未対応
- **メール送信機能** — アプリ全体でメール送信機能そのものが未実装
- **業者別／現場別の支払台帳レポート** — 集計 API はあるが UI と出力機能がない
- **資金繰りダッシュボード** — 期日別の「手形支払予定」「振込予定」の時系列ビューがない
- **支払額の訂正履歴** — 請求額と支払額の差分・訂正理由の記録機能がない
- **支払単位の確定ロック** — 現状は決算確定時のみロック（個別支払の確定ロック不可）
- **工事台帳・実行予算の PDF/Excel 出力** — 画面表示のみ、印刷帳票として出力不可

### 経理運用を成立させるための追加実装（優先度案）

| 優先度 | 機能 | 備考 |
|--------|------|------|
| 高 | 請求書 PDF の添付機能 | 既存の添付ファイル基盤を流用可。査定モーダルから即プレビューできる導線を追加 |
| 高 | 全銀フォーマット出力 | 確定済み振込分を FB データ(.txt) でダウンロード。経理が銀行へアップロード |

| 中 | 資金繰りダッシュボード | 期日別に「手形」「振込」の支払予定を時系列表示 |
| 中 | 業者別支払台帳画面 | 集計 API は既存。UI を追加するだけで月次・期別ビューが作れる |

## 会計年度（期）

- 1期 = 1951年度（1951年10月〜1952年9月）
- 10月1日に期が切り替わる
- 例：2026年2月 → 75期、2026年10月 → 76期

## セットアップ

1. Google スプレッドシートを作成し、`Code.gs` 内の `SPREADSHEET_ID` を設定
2. `OUTPUT_CONFIG` 内のテンプレートファイルID・出力フォルダIDを設定
3. GASエディタで `setupNewFeatureSheets()` を実行してシートを初期化
4. 必要に応じて `insertAllData()` でダミーデータを投入
5. GASエディタから「デプロイ」→「新しいデプロイ」→ Webアプリとしてデプロイ

## 更新履歴

### 契約協議書PDF出力に「出力日付-枝番」スタンプを追加

契約協議書のPDF出力時に、PDF1ページ目の右上へ「出力日付-枝番」（例: `20260428-1`）を自動で印字する機能を追加。あわせて、UIから直接PDFを出力できる導線を新設。

- **`Code.gs`**：
  - `ensurePdfExportLogSheet()` — 「PDF出力履歴」シートを自動作成
  - `getOrIssuePdfSequenceNo(contractId, department)` — 当日同契約はすでに発番済みの枝番を再利用、新規なら +1 で発番（`LockService` で排他制御）
  - `updatePdfExportLogFileName(seq, fileName)` — 履歴シートにファイル名を後追記
  - `exportContractToPdf()` — `enrichedData['出力番号']` を注入してプレースホルダー置換に乗せ、ファイル名にも `_20260428-1` を含める
  - `exportContractSmallToPdf()` を新設（小口契約協議書のPDF版＋枝番スタンプ）
- **`index.html`**：
  - `GasApi.exportContractSmallToPdf` ラッパー、`exportContractPdfSmall` メソッドを追加
  - `contract-form` に `📕 大工事用PDF出力` `📘 小口用PDF出力` ボタンを追加
- **テンプレート（手動作業）**：大工事用 / 小口用テンプレート両方に `<<[出力番号]>>` プレースホルダーセルを「印刷領域の1ページ目右上」に配置する必要あり
- 履歴は `SPREADSHEET_ID` 内に自動生成される `PDF出力履歴` シートに記録（出力日付 / 部署 / 契約ID / 枝番 / 出力番号 / 初回出力日時 / 最終出力日時 / 出力回数 / ファイル名）

### 協力会社・顧客マスタの 100 件制限を撤廃（軽量API + 遅延ロード方式）

協力会社マスタが 100 件を超えた環境で「注文決定業者名サジェストに該当業者が出ない」「マスタ画面で 101 件目以降が表示されない」不具合を修正。

- **`Repository.gs`**：`VendorRepository.search` / `CustomerRepository.search` の `slice(0, 100)` 制限を撤廃。全件返却に変更
- **`Code.gs`**：サジェスト用の軽量 API `getVendorsLight()` / `getCustomersLight()` を新設（必要最小限のフィールドのみ返す）。`initializeApp()` は軽量版を使用するよう切替
- **`index.html`**：
  - `GasApi.getVendorsLight()` / `getCustomersLight()` を追加
  - `loadMasters()` を軽量版呼び出しに変更
  - フル版を取得する `loadCustomersFull(force)` / `loadVendorsFull(force)` を追加し、マスタ管理画面入域時にバックグラウンドで取得
  - マスタ保存後の再取得をフル版に切替（編集中画面に変更が即時反映されるよう）
  - 状態フラグ `_customersFullLoaded` / `_vendorsFullLoaded` で多重ロード防止

### 入力フィールドの文字色を明示・Chrome autofill のグレー化を抑止

`.form-control` で文字色を継承任せにしていたため、Chrome の autofill が効いた入力欄（注文決定業者名等）で文字が薄く表示される事象を修正。

- `.form-control` に `color: var(--text-primary)` を明示
- `:-webkit-autofill` 系セレクタで `-webkit-text-fill-color` と `box-shadow inset` を上書きし、autofill による文字色グレー化・背景の黄ばみを抑止
