工事管理システム ユーザーマニュアル

本マニュアルでは、Google Apps Script で構築された建設業向け工事管理Webアプリの「できること」と「使い方」を画面別にまとめています。日々の業務でつまづいた際のリファレンスとしてご活用ください。

1システム概要

本アプリは、案件の発生から契約・発注・請求・支払まで、工事業務に関わる一連のデータを一元管理するためのWebアプリケーションです。 データの保存先は Google スプレッドシートで、出力ドキュメントは Google ドライブに自動保存されます。

このシステムでできること

📁 案件・工事の管理

案件の登録から進捗追跡、論理削除、ステータス別の集計まで対応。

📝 契約協議書

作成・承認フローに加え、Excel/PDF出力（枝番自動採番）に対応。

📦 発注・見積

明細の複数行管理、注文書/注文協議書の自動生成。

💰 請求・入金・支払

請求書発行、入金履歴、支払承認・実行記録、手形比率自動適用。

📊 決算集計・予算

月次・期別の集計、案件別実行予算の実績管理。

📅 スケジュール

ガントチャート、マイルストーン、カレンダーで進捗を可視化。

🏢 マスタ管理

顧客・協力会社・社員のマスタ管理（軽量APIで100件超もOK）。

🛠 運用支援

CSV取込、アーカイブ復元、操作履歴、添付ファイル連携。

用語

用語意味
案件引合・商談から契約に至るまでの単位。
工事契約後に発生する実際の施工単位。1案件に複数工事がぶら下がることもある。

2ログインと権限

WebアプリのURLにアクセスすると、Googleアカウントで自動認証されます。社員マスタに登録されているメールアドレスと一致したアカウントでのみ操作可能です。

ロール一覧

ロールレベルできること
readonly1各画面の閲覧のみ。データの作成・編集はできません。
staff2自部門の案件・契約・発注などのCRUD操作。
manager3承認権限、削除権限、アーカイブ管理。
admin4マスタ管理、完全削除、すべての機能。
ヒント 画面右上のユーザー名の横にロールが表示されます。ボタンが押せない / 入力欄がグレー化している場合はロール不足の可能性があります。

3業務全体の流れ

本システムは、以下のような大きな流れで業務データが連結されていきます。

  1. 案件登録：引合や商談が発生したら「案件管理」で新規作成。
  2. 契約協議書作成・承認：契約条件を入力し、上長が承認。承認後は編集ロックがかかります。
  3. 工事登録：承認済の契約協議書から工事を自動登録。追加工事にも対応。
  4. 発注：協力会社への発注情報を登録し、必要に応じて注文協議書 / 注文書を出力。
  5. 請求 / 入金：顧客への請求書を発行し、入金状況を記録。
  6. 支払：協力会社への支払を作成・承認・実行。手形比率は協力会社マスタから自動適用。
  7. 工事台帳・決算：工事台帳に契約・発注・請求・支払・予算が集約され、月次/期別で集計。

4案件・工事管理

案件管理

商談中の案件から契約済の案件まで、ステータスごとに切替えて確認できます。

  • 一覧画面のフィルタで「進行中 / 商談中 / 契約済 / 完了」を切替
  • 「+ 新規作成」から案件を登録
  • 削除は論理削除で行われ、配下の工事・契約・発注などにカスケード反映
  • 削除した案件はアーカイブ画面から復元可能

工事管理

契約済の案件にひも付く施工単位として工事を管理します。進捗率の更新やロック（編集禁止）も可能です。

5契約協議書

作成と承認

契約協議書フォームは「基本情報 → 発注者情報 → 工事情報 → 担当者情報」の順に上から下へ入力する導線になっています。承認済になると編集ロックがかかり、画面上部に黄色いバナーが表示されます。

  1. 左メニューから「契約協議書」を開き「+ 新規作成」をクリック
  2. 各タブを上から順に入力（顧客住所は顧客マスタから自動取得）
  3. 必要なら「複写して新規作成」で既存契約をベースに新規作成（複写であることが分かるよう「（複写）」表示が付与されます）
  4. 保存後、上長が「承認」操作。承認以降は編集不可になり、案件・工事の自動登録が可能になります

Excel / PDF 出力（枝番付き）

契約フォーム右上の出力ボタンから、テンプレートに値を埋め込んだExcel / PDFを生成し、Driveの所定フォルダに保存できます。

ボタン出力形式枝番スタンプ
📄 大工事用出力大工事（ソリューション）テンプレートExcel (.xlsx)—
📋 小口用出力小口契約テンプレートExcel (.xlsx)—
📕 大工事用PDF出力 NEW大工事（ソリューション）テンプレートPDFあり（右上）
📘 小口用PDF出力 NEW小口契約テンプレートPDFあり（右上）

枝番（出力番号）の仕様

  • 形式：YYYYMMDD-N（例：20260428-1）
  • 当日 1番目に出力した契約から順に -1, -2, -3... と採番
  • 同じ契約協議書を同日に再出力した場合は同じ枝番をそのまま再利用（番号は増えません）
  • 別契約を当日中に出力すると +1 で発番
  • 翌日になると -1 から再スタート
  • 同時押下による重複発番は LockService で防止

発番の履歴

発番状況はスプレッドシート内の「PDF出力履歴」シートに自動記録されます（初回出力時に自動作成）。各行には次の項目が保存されます。

  • 出力日付 / 部署 / 契約ID / 枝番 / 出力番号
  • 初回出力日時 / 最終出力日時 / 出力回数
  • 最終ファイル名
テンプレート側の準備（管理者向け・1回だけの初期設定） PDFの右上に枝番が印字されるためには、Google Drive上のテンプレートに <<[出力番号]>> というプレースホルダーセルを「印刷領域の1ページ目右上」に置く必要があります。
対象テンプレート：大工事用 / 小口用 の両方
フォントサイズや右寄せはテンプレート側のセル書式で調整してください。
右上に何も出ない場合
  1. テンプレートのセルに <<[出力番号]>> が入っているか確認（記号は半角）
  2. 印刷プレビューで該当セルが1ページ目の右上領域に入っているか確認
  3. 「PDF出力履歴」シートに行が追加されているか確認（追加されていれば発番ロジックは動作中）

6発注管理

協力会社への発注を作成・編集します。複数明細の見積もりや、注文書 / 注文協議書のドキュメント出力にも対応しています。

  • 発注フォームから複数の見積明細を一括登録
  • 注文協議書（社内回覧用）と注文書（社外提出用）を Google ドキュメントテンプレートから生成
  • 「複写」機能で既存発注を雛形に新規作成可（複写時は「（複写）」表示）
  • 発注確定時に支払データを自動生成（支払条件は協力会社マスタの手形比率・サイトを参照）

7請求・入金

顧客への請求書発行と、入金履歴の管理を行います。

  • 案件・工事を選択して請求書を作成
  • ステータス：下書き → 発行済 → 入金済
  • 入金履歴は1請求に対して複数件記録可能（分割入金対応）

8支払管理

協力会社への支払を作成・承認・実行する画面です。

  • 発注確定時に自動生成された支払データを編集・確定
  • 承認フロー：作成 → 承認待ち → 承認済 → 支払実行
  • 手形比率・サイト等は協力会社マスタから自動適用
  • 支払履歴を一覧で確認可能

9工事台帳

工事1件ごとに、契約・発注・請求・支払・予算データを集約表示します。完了工事の振り返りや原価分析に利用します。

  • 工事ごとのステータス管理（着工前 / 進行中 / 完了 など）
  • 関連する契約・発注・請求・支払が1画面に集約
  • 予算と実績の差異を確認可能

10実行予算

案件・工事ごとの予算を登録し、実績（発注・請求・支払）との差異を集計します。

  • 大科目 / 中科目 / 小科目の階層で予算を入力
  • 実績は発注・請求・支払シートから自動集計
  • 計画と実績の対比で着地予測を確認

11スケジュール

ガントチャート

工事の開始日 / 完工日をバーで表示。進捗状況をひと目で把握できます。

マイルストーン管理

  • 予定日と実績日を別々に入力
  • 予定日を過ぎても実績が未登録の場合は遅延として自動検知

カレンダー

契約期日や工事の開始 / 完了日を月単位のカレンダーに表示します。

12決算集計

月次・期別の集計を一覧で確認できます。

  • 期は10月始まり（1期 = 1951年度）
  • 月別集計は1ヶ月ごとの請求・支払・粗利を表示
  • 期別集計は12ヶ月分の累計を表示

13マスタ管理

admin ロールのみアクセス可能です。

顧客マスタ

  • 顧客の基本情報（住所・連絡先など）
  • 契約協議書の発注者欄に住所を自動反映

協力会社マスタ

  • 協力会社情報＋支払条件（手形比率・サイト）
  • 発注時に自動的に支払条件が適用される

社員マスタ

  • 社員のメールアドレスとロールを管理
  • このシートに登録された社員のみアプリにアクセス可
軽量API + 遅延ロード方式 顧客・協力会社マスタが100件を超えても全件表示・検索できるよう、初期表示は軽量API、マスタ画面に入った瞬間にバックグラウンドでフルデータを取得する設計になっています。100件目以降が出ない事象は解消済みです。

14CSV / アーカイブ / 操作履歴

CSVインポート

各シートに対してCSVから一括取込ができます。ヘッダ行をシートのカラム名と一致させてください。

アーカイブ

論理削除されたデータの一覧です。manager 以上で「復元」または「完全削除」が可能です。

操作履歴（監査ログ）

主要な操作（作成・編集・削除・承認・出力など）が「監査ログ」シートに自動記録されます。トラブル時の調査や改善にご活用ください。

添付ファイル

各レコードにGoogle Drive上のファイルを添付できます。プレビューはモーダルで開き、PDF・画像はその場で閲覧可能です。

15よくある質問

Q. PDFの右上に枝番が出ません

A. テンプレート側にプレースホルダーが入っていない可能性が最も高いです。契約協議書のExcel/PDF出力セクションの手順で <<[出力番号]>> セルがあるか確認してください。

Q. 同日に同じ契約をPDF出力したら枝番が増える？

A. 増えません。契約協議書ごとに当日初出力時に確定し、再出力では同じ番号が付きます。別契約を出力すると +1 採番されます。

Q. 「協力会社マスタで業者が見つからない」

A. 旧来の100件制限は撤廃済みです。それでも見つからない場合はマスタ管理画面で実在を確認してください。サジェストは入力中の文字でフィルタされます。

Q. 削除したデータを戻したい

A. manager 以上のロールでアーカイブ画面から「復元」してください。完全削除されたデータは戻せません。

Q. 入力欄の文字色が薄い（autofill問題）

A. 修正済みです（Chrome の autofill 状態でも標準色で表示されます）。表示が崩れている場合はキャッシュクリアやハードリロードをお試しください。

Q. ボタンが押せない / 入力欄がグレーアウト

A. ロール不足、または契約協議書が承認済みのため編集ロックがかかっている可能性があります。画面上部のバナーやロール表示を確認してください。
